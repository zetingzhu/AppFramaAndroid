# XTrendspeed 居住国家与城市选择页 - PRD 测试用例库

本文档为基于《居住国家/地区》和《城市页》的 UI 视觉稿与交互文档提取的结构化测试用例。此格式专为 LLM 及 MCP (Model Context Protocol) 自动化测试代理设计，以方便机器读取与逻辑推断。

---

## 测试模块 1：居住国家与国籍页 (Country & Nationality Page)

### TC-CN-001: 页面基础属性与数据排列验证
- **前置条件**: 进入“居住国家/地区”或“国籍”选择页面。
- **测试步骤**:
  1. 检查页面 Title。
  2. 观察国家列表项是否带有国旗 Icon。
  3. 检查具有字母体系语言的列表排序逻辑。
  4. 检查无字母顺序规则的语言列表。
- **预期结果**:
  1. Title 必须精确匹配为 “居住国家/地区” 或 “国籍”。
  2. 国家项必须展示对应国旗 icon。
  3. 字母语言：严格按首字母升序排列；首字母相同时按第二个字母排序。
  4. 非字母语言：展示无特定顺序规则。

### TC-CN-002: 单行点击选中逻辑
- **前置条件**: 列表已正常加载。
- **测试步骤**: 点击任意一个国家整行（包含文本或空白处）。
- **预期结果**: 自动退出当前页面，并向上一级页面返回并显示对应的国家名称。

## 测试模块 2：右侧字母索引栏 (Alphabet Index Bar - 全局通用)

### TC-IDX-001: 滑动联动高亮
- **前置条件**: 页面数据已加载，且语言支持字母排序。
- **测试步骤**: 页面向下滑动，使以 `B` 开头的第一个选项停留在页面顶部可视区。
- **预期结果**: 右侧字母索引栏的 `B` 字母自动变为**高亮状态**，其他字母置于暗色状态。且该高亮状态直到被下一字母分类替换前持续保持。默认状态下，顶部首字母应当高亮。

### TC-IDX-002: 点击按扭定位
- **前置条件**: 页面处于静止状态。
- **测试步骤**: 点击右侧字母索引栏中的任意非当前高亮字母（例如 `M`）。
- **预期结果**: 
  1. 点击后字母 `M` 立即高亮。
  2. 左侧列表瞬间将以 `M` 开头的分组滑动对齐展示到页面最顶部。

### TC-IDX-003: 无字母排序语言降级处理
- **前置条件**: 系统语言/数据被设置为无字母顺序的语言（如特定中东语言等）。
- **测试步骤**: 观察并尝试点击右侧字母索引。
- **预期结果**: 索引顶部仅展示默认符号 `#`，且整个右侧字母索引栏**不可被点击**（无任何滑动联动响应）。

## 测试模块 3：顶部搜索栏交互 (Top Search Bar - 全局通用)

### TC-SRCH-001: 搜索框激活与还原
- **前置条件**: 搜索框处于默认空闲状态。
- **测试步骤**: 
  1. 点击顶部搜索框。
  2. 点击右侧出现的【取消】按钮。
- **预期结果**:
  1. 点击搜索框后：软键盘拉起，光标在输入框内闪烁，输入框外部右侧出现【取消】按钮。提示词（“搜索国家/地区”或“搜索国籍”或“搜索城市”）正常显示。
  2. 点击【取消】后：关闭软键盘，搜索区域还原为未点击的原始状态（【取消】按钮隐藏）。

### TC-SRCH-002: 内容输入与清除联动
- **前置条件**: 焦点在搜索框且键盘已弹起。
- **测试步骤**:
  1. 键入任意一个字符 `A`。
  2. 点击输入框内侧出现的【清除】(X) 按钮。
- **预期结果**:
  1. 键入字符后：原本文本提示词隐藏；输入框右侧（取消按钮的左边）展示【清除】按钮；列表立刻响应过滤，仅显示符合此输入字符筛选的选项。
  2. 点击【清除】后：输入框内文本瞬间清空，列表展示恢复全量初始数据状态。

## 测试模块 4：城市选择页专属交互 (City Page Exclusives)

*(涵盖上述所有通用的排序、搜索、字母索引逻辑，以下为独有逻辑)*

### TC-CTY-001: 底部手动录入框激活体验
- **前置条件**: 进入城市选择页，页面静止。
- **测试步骤**: 
  1. 观察底部悬浮输入框。
  2. 点击该底部输入框区域。
- **预期结果**:
  1. 默认展示提示语：“无选项可手动填写”。
  2. 点击后：拉起系统软键盘，光标闪烁，此时提示语隐藏。

### TC-CTY-002: 底部手动录入框状态机与提交
- **前置条件**: 底部手动输入框已获取焦点。
- **测试步骤**: 
  1. 键入任意合法内容（如："测试城市"）。
  2. 观察控件可见性。
  3. 清除内容（使其再次变为空）。
  4. 重新键入内容并点击【确定】按钮。
- **预期结果**:
  1. 有内容时：输入框内展示【清除】按钮；输入框外部右侧出现【确定】按钮。
  2. 无内容时：隐藏【确认】按钮，仅保留空状态。
  3. 点击【确定】后：页面退出，返回至上级的地址国籍信息表单页，并在对应的数据横线上回显展示在此手动输入的城市信息。

### TC-CTY-003: 清空策略与异常交互阻断
- **前置条件**: 处于城市选择页。
- **测试步骤**:
  1. 在底部手动输入框中输入部分保留文字，随后点击屏幕非底部弹窗的其他任何位置。
  2. 彻底退出城市页，随后从上一级重新进入该页面。
- **预期结果**:
  1. 点击其他位置：应当退出“手工编辑此区域”的状态（收起键盘，丧失焦点，返回正常的城市列表选择交互图层）。*(注: PRD 描述“退出该页面”有语意歧义，合理实现为退出输入状态)*
  2. 重新进入页面：底部输入框必须处于全新的空状态（不保留上次留存的任何文本输入内容，每次进入必须清空刷新）。
