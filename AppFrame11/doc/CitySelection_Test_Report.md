# 城市选择页功能测试差异报告 (CitySelectionActivity)

**文档状态**: 待优化项汇总
**基于文档**: `xtrendspeed_test_cases.md`
**测试对象**: `CitySelectionActivity.kt` 及相关组件

---

## 1. 列表展示与视觉 (TC-CN-001)
*   **差异点**: **缺失 Icon 展示**。
    *   **预期**: 国家/城市列表项必须展示对应的 Icon（国旗或城市标识）。
    *   **现状**: `CityAdapter.kt` 中的 `ContentViewHolder` 仅包含一个 `TextView`，未定义或加载任何 `ImageView`。
*   **差异点**: **搜索后的数据结构破坏**。
    *   **现状**: `filterCities` 方法执行后，列表会丢失所有 `Header`（字母索引头），变成纯列表。虽然满足搜索结果展示，但与初始列表的 A-Z 分类结构不一致，导致搜索结果缺乏上下文归属感。

## 2. 右侧字母索引栏 (TC-IDX-003)
*   **差异点**: **缺失无序语言的降级逻辑**。
    *   **预期**: 当数据无字母顺序规则时（如某些非字母语言），索引栏应仅展示 `#` 且不可点击。
    *   **现状**: `AlphabetIndexView.kt` 始终硬编码显示 A-Z 索引，且没有针对非字母语言数据的隐藏或禁用逻辑。

## 3. 顶部搜索栏交互 (TC-SRCH-001/002)
*   **差异点**: **取消搜索的体验不完整**。
    *   **现状**: 点击“取消”按钮会清空文本并隐藏键盘，但并未将 `recyclerView` 滚动回顶部或恢复到搜索前的浏览位置。

## 4. 底部手动录入框 (TC-CTY-001/002/003)
*   **差异点**: **提示语隐藏时机不符合 PRD**。
    *   **预期**: “点击后（获取焦点）...此时提示语（Hint）隐藏”。
    *   **现状**: 目前使用的是系统的 `android:hint`，默认逻辑是“开始输入”才隐藏。PRD 要求“点击即隐藏”。
*   **差异点**: **点击其他位置退出编辑状态的覆盖范围不足** (TC-CTY-003.1)。
    *   **预期**: 点击页面其他任何位置应当收起键盘并让输入框丧失焦点。
    *   **现状**: 当前仅在 `recyclerView` 上设置了 `OnTouchListener`。如果用户点击顶部的 TitleBar 或页面背景空白处，输入框仍会保持焦点和键盘弹出状态。
*   **差异点**: **页面重入清理逻辑缺失** (TC-CTY-003.2)。
    *   **预期**: 重新进入页面时，底部输入框必须处于全新的空状态。
    *   **现状**: 代码中未在 `onStart` 或 `onResume` 中显式清空 `etManualInput`。

---

### 建议修改计划:

1.  **UI 升级**: 修改 `CityAdapter.kt` 的布局，引入 `ImageView` 并支持加载图标资源。
2.  **搜索逻辑重构**: 修改 `filterCities`，在搜索结果中动态构建包含 `Header` 的新列表，以保持 A-Z 结构。
3.  **焦点与键盘管理**: 将点击关闭键盘的逻辑从 `recyclerView` 移动到 Root View 层级，并利用 `OnFocusChangeListener` 实现“点击即隐藏 Hint”。
4.  **组件健壮性**: 为 `AlphabetIndexView` 增加 `setData(letters: List<String>)` 接口，支持根据数据源动态调整显示的字母。
