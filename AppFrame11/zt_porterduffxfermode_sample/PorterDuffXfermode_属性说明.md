# PorterDuffXfermode 混合模式完全指南

`PorterDuffXfermode` 是 Android 中用于处理图形混合（图像合成）的核心类。它允许你通过指定不同的 `PorterDuff.Mode`，将源图形（Source，简称 SRC，即上层新绘制的图形）与目标图形（Destination，简称 DST，即下层已存在的图形）按照特定的数学规则进行像素级别的混合。

---

## 核心概念说明

在使用 `PorterDuffXfermode` 之前，必须理解以下几个核心概念：

1. **DST (Destination / 目标图)**：下层图，即已经存在于画布（Canvas）上的内容。
2. **SRC (Source / 源图)**：上层图，即当前正准备通过画笔（Paint）绘制上去的新内容。
3. **离屏缓冲 (SaveLayer)**：
   - **非常重要**：使用 Xfermode 时，强烈建议使用 `canvas.saveLayer()` 创建一个具有透明背景的独立离屏图层。
   - **原因**：如果不使用离屏缓冲，Xfermode 可能会将整个屏幕的底色（比如 Activity 的白色背景）也当做 DST 参与混合运算，从而导致不可预期的黑色或无效效果。
4. **透明度 (Alpha)**：大部分 PorterDuff 模式都严格依赖像素的 Alpha 通道进行计算。源图形的透明度变化会直接影响最终的合成效果。

---

## 混合模式分类详解

PorterDuff 的 18 种模式可以大致分为四类：清除、Alpha 蒙版、覆盖与裁剪、色彩混合。

### 一、清除模式 (Clear)
直接擦除像素，通常用于实现橡皮擦效果。

| 模式 | 公式定义 | 效果说明 | 适用场景 |
| :--- | :--- | :--- | :--- |
| **CLEAR** | `[0, 0]` | 清除源与目标重叠的区域。计算结果的 Alpha 和 Color 均为 0。不受源图形透明度影响。 | 强力橡皮擦、完全掏空背景。 |

### 二、Alpha 蒙版类 (Alpha Compositing)
这类模式主要用于裁剪、遮罩、保留交集等操作。它们对透明度非常敏感，常用于圆角头像、倒影、边缘羽化等。

| 模式 | 效果说明 | 适用场景 |
| :--- | :--- | :--- |
| **SRC_IN** | 仅保留 **源与目标相交** 部分。交集处显示源图形。若目标半透明，源图形也会相应变透明。 | 将图片裁剪为特定形状（如圆角头像，DST画形状，SRC画图片）。 |
| **DST_IN** | 仅保留 **目标与源相交** 部分。交集处显示目标图形。 | **图片倒影、边缘羽化淡出**（利用渐变SRC保留部分DST）。 |
| **SRC_OUT** | 仅保留 **源中不与目标重叠** 的部分。交集区域被掏空。 | 绘制内部有镂空图案的图形。 |
| **DST_OUT** | 仅保留 **目标中不与源重叠** 的部分。交集区域被源图形“擦除”。 | **橡皮擦、刮刮乐效果、遮罩掏空**。 |

#### 💡 进阶原理解析：当 DST_IN 和 DST_OUT 遇上 LinearGradient (线性渐变)

在实际开发中（如实现 `TopTransparentFrameLayout` 边缘淡出效果），我们经常使用从不透明到透明的 `LinearGradient` 作为源图形（SRC），完全覆盖在目标图形（DST）上。
此时，`DST_IN` 和 `DST_OUT` 会呈现出**方向相反的渐变效果**，其本质是由它们的计算公式决定的：

假设渐变源图形（SRC）为**从上到下：不透明(Alpha=100%) $\rightarrow$ 完全透明(Alpha=0%)**。

**1. DST_IN (正向蒙版 / 保留相交)**
*   **规则**：目标透明度 = 目标自身透明度 $\times$ **源图形的透明度**。
*   **顶部计算**：SRC 透明度 100%。DST 透明度 = 100% $\times$ 100% = **100% (完全显示)**。
*   **底部计算**：SRC 透明度 0%。DST 透明度 = 100% $\times$ 0% = **0% (完全透明/消失)**。
*   **视觉效果**：目标图形**上边清晰、下边淡出消失**。常用于列表顶/底部滚动渐隐。

**2. DST_OUT (反向蒙版 / 擦除相交)**
*   **规则**：目标透明度 = 目标自身透明度 $\times$ **(1 - 源图形的透明度)**。
*   **顶部计算**：SRC 透明度 100%。DST 透明度 = 100% $\times$ (1 - 100%) = **0% (被完全擦除)**。
*   **底部计算**：SRC 透明度 0%。DST 透明度 = 100% $\times$ (1 - 0%) = **100% (完全显示)**。
*   **视觉效果**：目标图形**上边消失、下边清晰**。相当于用一块上面用力、下面轻柔的渐变橡皮擦进行擦拭。

### 三、覆盖与裁剪模式 (Over & Atop)
用于控制哪个图层在上，或者利用底层约束上层的边界。

| 模式 | 效果说明 | 适用场景 |
| :--- | :--- | :--- |
| **SRC** | 仅保留源图像，目标图像完全被丢弃。 | 强制替换某块区域的内容。 |
| **DST** | 仅保留目标图像，源图像不起任何作用。 | 锁定图层，保护已有内容不被修改。 |
| **SRC_OVER** | **默认模式**。源覆盖在目标之上，正常叠加。 | 正常的绘制行为，后画的在上面。 |
| **DST_OVER** | 目标覆盖在源之上。即源图形被画在目标图形的“背后”。 | 在已有图形背后添加背景或光晕。 |
| **SRC_ATOP** | 源绘制在目标之上，但**源超出目标的部分被裁剪丢弃**。 | 保证新绘制的内容绝对不会超出原有图形的边界。 |
| **DST_ATOP** | 目标绘制在源之上，但**目标超出源的部分被裁剪丢弃**。 | 较少使用。 |
| **XOR** | 异或模式。仅显示双方**非重叠**区域，交集部分变透明。 | 制作环形或相交镂空的特殊几何图形。 |

### 四、色彩混合模式 (Color Blending)
这类模式不仅处理 Alpha 通道，还对 RGB 颜色通道进行复杂的数学运算，常用于图像处理（类似 Photoshop 中的图层混合模式）。

| 模式 | 效果说明 | 适用场景 |
| :--- | :--- | :--- |
| **DARKEN** | 取两图层重叠区域对应像素的**较暗**颜色。 | 压暗画面、去除纯白背景保留深色内容。 |
| **LIGHTEN** | 取两图层重叠区域对应像素的**较亮**颜色。 | 提亮画面、去除纯黑背景保留亮色内容。 |
| **MULTIPLY** | 正片叠底。源与目标颜色通道相乘，结果总是变暗。 | 增加阴影细节、模拟多层彩色玻璃叠加。 |
| **SCREEN** | 滤色。源与目标的补色相乘，结果总是变亮。 | 添加高光、光斑、模拟发光效果。 |
| **ADD** | 颜色通道直接相加，并截断在 255 (白色)。 | 产生极强的曝光和高亮效果。 |
| **OVERLAY** | 叠加。结合了 Multiply 和 Screen，暗处更暗，亮处更亮。 | 增强图像对比度，给黑白图片上色。 |

## 最佳实践与注意事项

1. **硬件加速问题**：
   部分混合模式在开启硬件加速的设备上可能表现异常或不支持。如果在自定义 View 中发现 Xfermode 效果不对，可以尝试在该 View 上关闭硬件加速：
   `setLayerType(View.LAYER_TYPE_SOFTWARE, null)`
2. **背景干扰**：
   始终牢记 Xfermode 是对**整个目标画布**上已存在的像素进行计算。如果你的 Activity 有白色背景，且没有使用 `saveLayer` 隔离，白色背景也会参与计算，导致 `DST_IN` 或 `CLEAR` 出现黑色块。
3. **性能消耗**：
   `saveLayer` 会创建一个离屏的 Bitmap 进行渲染，这是一个比较耗费内存和 GPU 资源的操作。在频繁重绘的场景（如动画、滑动列表）中，应尽量减小 `saveLayer` 的区域大小，只在需要混合的局部使用。
